# 答案之书 · Healing Answers

这是一个前端静态网页版本的「答案之书」，用户可以在页面中央写下自己的问题，点击「翻开答案之书」，随机翻到一条答案（支持普通答案与「好话/疗愈」答案两种分组）。  
在获得答案后，用户还可以绑定自己在 Anthropic / OpenAI / Google Gemini / DeepSeek 的 API Key，调用对应大模型，为当前问题和答案生成一段更细致的深度解答。

> 重要说明：本项目**不会提供也不会代管任何 API 服务**，也**不会上传或存储任何用户的 API Key**；所有 Key 仅保存在用户本地浏览器的 `localStorage` 中，用户可随时在页面上点击「清除保存的 Key」将其清除。

## 快速开始

- **方式一：直接双击打开**
  - 直接用浏览器打开 `index.html` 即可使用（推荐 Chrome / Edge / Safari）。

- **方式二：本地起一个静态服务器预览**

  ```bash
  npm install
  npm run start
  ```

  然后在浏览器访问终端输出的本地地址（一般是 `http://localhost:3000` 或 `http://localhost:5000` 等，取决于 `serve` 工具）。

## 填入你自己的答案库

当前项目中的答案只是一小段占位示例。你可以在 `app.js` 顶部找到如下代码，并将其替换为自己的完整答案库：

```js
const ANSWER_LIBRARY = {
  normal: [
    // 普通答案
  ],
  healing: [
    // 好话 / 疗愈向答案
  ]
};
```

- **`normal`**：普通答案集合。
- **`healing`**：当勾选「疗愈状态」时，仅会从这里抽取答案；未勾选时，会从 `normal + healing` 的总集合中抽取。

你可以在准备好答案库之后，把对应数组替换进去即可。

## 页面结构与交互简介

- **问题输入区**
  - 中央输入框，限制在 **30 个字以内**。
  - 右侧有「疗愈状态」按钮，点亮后只会从「好话」组中抽取答案。
  - 下方按钮「翻开答案之书」会触发一本书样式的翻页动画，并随机展示一条答案。

- **深度解答**
  - 用户可以分别为 Claude / GPT / Gemini / DeepSeek 输入自己的 API Key。
  - 选择要使用的模型后，点击「获取深度解答」：
    - 前端会把「用户问题 + 随机翻到的答案」一起发送给选定的大模型。
    - 得到一段更加细致、偏疗愈风格的长文本回应。
  - 所有 Key 通过 `localStorage` 存在用户浏览器本地，并提供一键清除按钮。

## 安全与隐私

- 本项目没有后端，不会把任何数据上传到服务器。
- API 调用均在浏览器中直接向各家官方 API 服务器发起（请确保在 **可信环境** 下使用）。
- 如果你在 **公用或不安全的电脑** 上使用，请在离开前务必点击页面上的「清除保存的 Key」按钮。

## 自定义样式与动效

- 页面使用柔和的深色星空背景与渐变，搭配书页翻动动画，营造安静、温暖、疗愈的氛围。
- 你可以在 `style.css` 中根据喜好调整颜色、字体与动画效果。

欢迎在此基础上继续扩展，例如增加：

- 为每条答案添加标签、主题、权重等。
- 为深度解答区域增加「复制内容」按钮。
- 增加「历史记录」面板，保存每一次提问和答案（同样可以只保存在本地）。
